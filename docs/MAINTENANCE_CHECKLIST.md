# Maintenance Checklist

Checklist operasional untuk menjaga website tetap sehat setelah launch.

## 1) Weekly checks
- Buka homepage, Karya, Wawasan, dan Kontak dari mobile + desktop.
- Tes CTA utama: **Ajukan Kolaborasi** dan **Mulai Percakapan Proyek**.
- Cek 1–2 konten terbaru: typo, gambar, dan link internal.
- Cek analytics dasar (traffic dan interaksi CTA).

## 2) Monthly checks
- Audit konsistensi tag **Area / Ruang** pada konten baru.
- Evaluasi performa konten terbaik dan terlemah.
- Rapikan judul/excerpt agar lebih tajam bila diperlukan.
- Cek kembali halaman penting untuk kualitas mobile UX.

## 3) Setelah publish Karya baru
- Verifikasi thumbnail arsip Karya.
- Verifikasi halaman Studi Kasus terbaca rapi di mobile.
- Klik CTA WhatsApp/kontak pada halaman detail.
- Pastikan tag Area / Ruang sudah relevan.

## 4) Setelah publish Wawasan baru
- Verifikasi excerpt tampil benar di listing Wawasan.
- Verifikasi link ke Karya (jika ada) berjalan.
- Cek CTA diskusi proyek pada halaman detail Wawasan.

## 5) Setelah ubah CTA / info kontak
- Uji semua tombol CTA lintas halaman.
- Uji tautan WhatsApp/email yang diperbarui.
- Pastikan tidak ada tombol yang mengarah ke URL lama.

## 6) Setelah deploy di Vercel
- Cek halaman utama dan halaman dinamis (`/karya/[slug]`, `/wawasan/[slug]`).
- Cek `/robots.txt` dan `/sitemap.xml` dapat diakses.
- Cek 404 page dengan URL acak.

## 7) Setelah menambah banyak gambar
- Cek kecepatan loading halaman Karya/detail.
- Cek crop/komposisi thumbnail.
- Cek apakah urutan galeri sudah sesuai cerita visual.

## 8) Analytics check
- Pantau pageview dan tren kunjungan halaman inti.
- Pantau interaksi CTA utama.
- Catat konten mana yang mendorong percakapan/lead.

## 9) SEO check
- Pastikan title dan excerpt tiap konten baru jelas.
- Pastikan slug tidak berubah sembarangan.
- Pastikan konten baru terhubung secara internal (Karya ↔ Wawasan bila relevan).

## 10) Broken link check
- Klik manual link penting dari konten terbaru.
- Prioritaskan link CTA, link ke Studi Kasus, dan link halaman kontak.

## 11) Mobile check
- Uji halaman di viewport mobile.
- Pastikan teks terbaca, tombol mudah ditap, dan layout tidak “pecah”.

## 12) Backup/export note
- Simpan salinan teks final konten (judul, excerpt, narasi) di dokumen internal.
- Simpan aset gambar final di folder terstruktur (per proyek/per artikel).
- Untuk backup teknis database/storage, koordinasikan dengan developer.
